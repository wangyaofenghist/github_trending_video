# GitHub Trending 视频生成系统 - 产品使用文档

## 1. 产品概述

### 1.1 产品定位

GitHub Trending 视频生成系统是一款自动化内容生产工具，能够：
- 自动抓取 GitHub Trending 热门项目
- 使用 AI 大模型分析项目特点
- 自动生成视频文案
- 生成带有字幕和配图的短视频

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| **自动化** | 从抓取到视频生成全流程自动化，无需人工干预 |
| **智能化** | 使用 LLM 大模型进行项目分析和文案创作 |
| **批量化** | 支持并发处理多个项目，高效产出 |
| **可定制** | 支持多种大模型配置，灵活切换 |

## 2. 功能模块

### 2.1 项目抓取模块

**功能**: 从 GitHub Trending 页面抓取热门项目信息

**抓取内容**:
- 项目基本信息（名称、描述、语言、Stars、Forks）
- 项目 Topics 标签
- README 原始内容
- 项目链接

**操作方式**:
- 手动触发：在首页点击"立即抓取"按钮
- 定时任务：通过 cron 定时执行每日抓取

### 2.2 项目分析模块

**功能**: 使用 AI 分析项目 README，提取关键信息

**分析内容**:
- 项目用途和适用场景
- 核心功能特点（3-5 个）
- 安装命令
- 快速开始示例
- 官方文档链接

**深度分析**（可选）:
- 详细使用场景
- 开源团队信息
- 市场前景分析
- 技术栈解析
- 竞品分析

### 2.3 文案生成模块

**功能**: 基于项目分析结果，自动生成视频文案

**生成内容**:
- 视频标题
- 开场 Hook（吸引注意力的开头）
- 主体文案内容
- 核心亮点（Key Highlights）
- 行动号召（Call to Action）

### 2.4 视频生成模块

**功能**: 将文案转换为短视频

**输出内容**:
- 视频画面（根据文案内容生成）
- 字幕（待实现）
- TTS 配音（待实现）

### 2.5 图片管理模块

**功能**: 管理视频生成所需的图片素材

**图片分类**:
- 封面图（Cover）
- 内容图（包含多图/视频）
- 结尾图（End）

## 3. 操作流程

### 3.1 完整工作流

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  1. 抓取项目  │───>│  2. 分析项目  │───>│  3. 生成文案  │───>│  4. 生成视频  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       │                  │                  │                  │
   获取 trending      LLM 分析 README     LLM 创作文案        合成视频素材
   项目和 README      提取关键信息        结构化输出          输出 MP4
```

### 3.2 详细步骤说明

#### 步骤 1: 抓取项目

1. 访问系统首页
2. 点击"立即抓取"按钮
3. 系统自动访问 GitHub Trending 页面
4. 抓取 Top 25 项目信息并保存到数据库

#### 步骤 2: 分析项目

1. 进入审核页面或项目详情页
2. 点击"分析"按钮（浅层分析）或"深入分析"按钮
3. 系统调用 LLM API 分析 README 内容
4. 分析结果保存到数据库

#### 步骤 3: 生成文案

1. 在项目分析完成后
2. 点击"生成文案"按钮
3. 系统基于分析结果创作文案
4. 文案可在审核页面编辑和预览

#### 步骤 4: 加入审核队列

1. 文案生成完成后
2. 点击"加入审核"按钮
3. 系统创建视频生成任务
4. 任务状态变为"待审核"

#### 步骤 5: 审核与生成

1. 在审核页面查看文案和图片
2. 可编辑文案内容
3. 可上传/管理图片素材
4. 审核通过后，视频进入生成队列

#### 步骤 6: 查看视频

1. 视频生成完成后
2. 在视频库页面查看
3. 可在线播放或下载视频

## 4. 页面说明

### 4.1 首页 (`/`)

**功能**: 项目概览和操作入口

**内容**:
- 统计卡片（项目总数、已分析、有文案、视频完成）
- 操作按钮（立即抓取、进入审核、视频库）
- 项目列表（展示最新抓取的项目）

**状态标记**:
- ✓ 已分析（绿色）- 已完成基础分析
- ✓ 深度分析（紫色）- 已完成深度分析
- ✓ 有文案（蓝色）- 已生成视频文案
- ✓ 视频完成（绿色）- 视频已生成
- ✓ 待审核（黄色）- 视频等待审核
- 无视频（灰色）- 未生成视频

### 4.2 审核页面 (`/review`)

**功能**: 审核文案和管理素材

**内容**:
- 项目列表（按日期分组）
- 项目详情（文案编辑、图片管理）
- 操作按钮（生成视频、删除项目）

### 4.3 视频库页面 (`/videos`)

**功能**: 查看和管理已生成的视频

**内容**:
- 视频列表（缩略图、标题、状态）
- 操作按钮（查看视频、移除）

### 4.4 项目详情页 (`/projects/<id>`)

**功能**: 查看项目完整信息

**内容**:
- 项目基本信息
- 分析结果
- 深度分析（如有）
- 文案内容
- 图片素材
- 视频任务状态

## 5. 删除功能

系统支持删除操作，可清理不需要的项目和视频：

### 5.1 删除视频任务

- 位置：视频库页面
- 操作：点击"移除"按钮
- 效果：删除视频任务和关联的视频文件

### 5.2 删除项目

- 位置：首页、审核页面
- 操作：点击"移除"按钮
- 效果：级联删除项目及其所有关联数据（分析、文案、视频、图片）

## 6. 注意事项

### 6.1 API Key 配置

系统需要配置至少一个大模型的 API Key：

- **推荐**: 通义千问（性价比高，中文支持好）
- **备选**: Anthropic Claude（高质量输出）
- **备选**: OpenAI GPT（国际项目适用）

### 6.2 视频生成说明

当前版本视频生成功能：
- ✅ 视频画面生成
- ⏳ TTS 语音合成（待实现）
- ⏳ 字幕烧录（待实现）

### 6.3 并发控制

系统支持并发调用 LLM API，可通过环境变量调整：

- `LLM_MAX_WORKERS`: 最大并发数（默认 5）
- `LLM_MAX_RETRIES`: 失败重试次数（默认 2）
- `LLM_REQUEST_TIMEOUT`: 单次请求超时（默认 120 秒）

## 7. 常见问题

### Q1: 抓取失败怎么办？
A: 检查网络连接，确认能访问 GitHub。如使用代理，确保代理配置正确。

### Q2: 分析失败怎么办？
A: 检查 LLM API Key 配置是否正确，确认账户余额充足。

### Q3: 视频无法播放怎么办？
A: 当前版本生成的视频没有音频轨道，这是已知问题，将在后续版本修复。

### Q4: 如何修改已生成的文案？
A: 在审核页面点击项目，进入文案编辑模式，修改后保存即可。
